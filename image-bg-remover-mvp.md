## 1. 项目概述

### 1.1 项目简介
一个基于 Web 的智能图片背景移除工具，用户上传图片后自动去除背景，生成透明背景图片。

### 1.2 核心价值
- **简单易用**：拖拽上传，一键处理
- **自动处理**：AI 智能识别主体，自动移除背景
- **即时反馈**：实时预览处理结果
- **免费开源**：无需注册，免费使用

### 1.3 目标用户
- 电商卖家（商品图背景处理）
- 社交媒体用户（头像、发帖图优化）
- 设计师（快速抠图辅助）
- 普通用户（证件照、生活照处理）

## 3. 技术架构

### 3.1 前端技术栈
- **框架**：React 18 + TypeScript
- **构建工具**：Vite
- **样式**：Tailwind CSS
- **组件库**：React Dropzone（上传）、Ant Design（基础组件）
- **HTTP 客户端**：Axios 或 fetch

### 3.2 后端技术栈
- **框架**：FastAPI（Python）
- **ASGI 服务器**：Uvicorn
- **背景移除**：rembg（U^2-Net 模型）
- **图片处理**：Pillow（PIL）
- **文件上传**：python-multipart

### 3.3 系统架构图

```
用户浏览器
    ↓
React 前端（上传图片）
    ↓
FastAPI 后端（接收文件）
    ↓
rembg 模型（处理背景）
    ↓
返回透明 PNG
    ↓
前端展示 + 用户下载
```

## 4. 功能详细说明

### 4.1 图片上传

**需求描述：**
- 支持点击按钮选择文件上传
- 支持拖拽图片到指定区域上传
- 支持常见图片格式：JPG、PNG、WEBP
- 文件大小限制：最大 10MB

**交互规范：**
- 上传前显示上传区域（虚线框、提示文字）
- 上传中显示进度条
- 上传成功后跳转到处理界面

**验收标准：**
- [ ] 支持 JPG、PNG、WEBP 格式
- [ ] 文件超过 10MB 时提示错误
- [ ] 拖拽上传正常工作
- [ ] 点击上传正常工作

### 4.2 背景移除

**需求描述：**
- 使用 rembg 库进行背景移除
- 自动识别图片主体并移除背景
- 处理时间 < 5 秒（常规尺寸）

**技术实现：**
```python
from rembg import remove
from PIL import Image

input_image = Image.open("input.jpg")
output_image = remove(input_image)
output_image.save("output.png")
```

**验收标准：**
- [ ] 处理后生成透明 PNG
- [ ] 主体保留完整
- [ ] 背景清理干净
- [ ] 处理时间合理（<10秒）

### 4.3 结果预览

**需求描述：**
- 显示原图和处理后的图片
- 支持对比查看（并排显示或切换）
- 支持棋盘格背景（展示透明区域）

**交互规范：**
- 默认显示处理后的结果
- 提供"查看原图"按钮切换
- 透明区域显示棋盘格图案

**验收标准：**
- [ ] 透明区域正确显示
- [ ] 原图和结果可以切换
- [ ] 图片缩放自适应屏幕

### 4.4 下载功能

**需求描述：**
- 一键下载处理后的透明 PNG
- 文件名保留原名前缀（如：`photo-bg-removed.png`）

**交互规范：**
- 点击"下载"按钮触发浏览器下载
- 下载完成后提示成功

**验收标准：**
- [ ] 下载的是 PNG 格式
- [ ] 文件名有意义
- [ ] 下载速度正常

### 4.5 错误处理

**需求描述：**
- 上传非图片格式时提示错误
- 处理失败时提示错误信息
- 网络错误时重试提示

**错误提示示例：**
- "不支持的文件格式，请上传 JPG、PNG 或 WEBP"
- "文件过大，请上传小于 10MB 的图片"
- "处理失败，请重试或更换图片"

## 5. 非功能性需求

### 5.1 性能要求
- 页面加载时间 < 3 秒
- 图片处理时间 < 10 秒（常规尺寸）
- 支持并发上传 5 个请求

### 5.2 兼容性
- 浏览器：Chrome/Edge（最新版）、Firefox（最新版）、Safari（最新版）
- 移动端：iOS Safari、Android Chrome

### 5.3 安全性
- 文件大小限制（防止拒绝服务攻击）
- 文件类型验证（后端二次校验）
- 不存储用户图片（处理完立即删除）

### 5.4 可用性
- 界面简洁直观
- 操作步骤不超过 3 步
- 有明确的操作提示

## 7. 部署方案

### 7.1 开发环境
- 前端：`npm run dev`（localhost:5173）
- 后端：`uvicorn app.main:app --reload`（localhost:8000）

### 7.2 生产环境（可选）
- **方案一**：Render / Railway（免费部署）
- **方案二**：阿里云 / 腾讯云服务器 + Nginx
- **方案三**：Docker + Docker Compose（一键部署）

### 7.3 环境变量
```

MAX_FILE_SIZE=10485760  # 10MB
ALLOWED_FORMATS=.jpg,.jpeg,.png,.webp
```
